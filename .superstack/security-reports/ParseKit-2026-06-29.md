## Executive Summary

ParseKit is a **local-first Tauri v2 desktop app** with no network-facing API, no auth layer, and no LLM integration. The security posture is appropriate for a menu-bar document converter: data stays on-device, the updater uses **minisign signature verification**, and signing keys are **gitignored**.

The highest-actionable issues are **defense-in-depth gaps on two unused/exposed Tauri IPC commands** (`save_error_report` path handling, `copy_file_to_clipboard` arbitrary read). No committed secrets were found. No CI/CD pipeline exists — releases rely on local `publish-release.sh`.

**Overall risk:** Low for typical end-user threat model; moderate for supply-chain / malicious-document parsing scenarios.

## Findings

### [MEDIUM] FINDING-001: `save_error_report` allows path traversal via `file_name`

**Confidence:** 9/10  
**Phase:** Phase 9 — OWASP A01 (Broken Access Control) / A05 (Injection)  
**Category:** OWASP A01  
**Location:** `src-tauri/src/lib.rs:732-746`

**Description:**  
`save_error_report` joins `dir` and `file_name` with `PathBuf::push` without sanitizing `file_name`. A value like `../../../tmp/evil.md` produces a path that escapes the intended output directory when normalized at write time.

**Exploit scenario:**
1. Attacker achieves arbitrary JS execution in the webview (e.g., compromised build dependency or future XSS).
2. Attacker calls `invoke('save_error_report', { dir: userOutputDir, fileName: '../../../.zshrc', contents: 'malicious' })`.
3. File is written outside the user's chosen output folder with the app's file permissions.

**Evidence:**
```rust
path.push(&file_name);
std::fs::write(&path, contents)?;
```
Verified: `PathBuf::push("../../../tmp/evil.md")` on base `/Users/foo/output` yields `/Users/foo/output/../../../tmp/evil.md`.

**Remediation:**
```rust
fn safe_file_name(name: &str) -> Result<String, String> {
    let base = std::path::Path::new(name)
        .file_name()
        .and_then(|s| s.to_str())
        .ok_or_else(|| "Invalid file name".to_string())?;
    if base.is_empty() || base == "." || base == ".." {
        return Err("Invalid file name".into());
    }
    Ok(base.to_string())
}
```
Use only the sanitized basename; reject paths containing `/` or `..`.

**Priority:** P1 (fix this sprint)

### [MEDIUM] FINDING-002: `copy_file_to_clipboard` exposes arbitrary file read over IPC

**Confidence:** 8/10  
**Phase:** Phase 9 — OWASP A01  
**Category:** OWASP A01  
**Location:** `src-tauri/src/lib.rs:969-971`

**Description:**  
Command reads any filesystem path via `std::fs::read` with no `validate_user_path` or permission check. It is registered in `invoke_handler` but **not called from frontend code** — dead attack surface.

**Exploit scenario:** Same as FINDING-001: webview JS execution → `invoke('copy_file_to_clipboard', { path: '/Users/victim/.ssh/id_rsa' })` → file bytes returned to attacker-controlled JS.

**Evidence:**
```rust
fn copy_file_to_clipboard(path: String) -> Result<Vec<u8>, String> {
    std::fs::read(&path).map_err(|e| format!("Failed to read file: {}", e))
}
```

**Remediation:** Remove the command if unused, or gate with `validate_user_path` and an allowlist (e.g., only files under user-selected output dirs).

**Priority:** P1 (fix this sprint)

### [LOW] FINDING-003: Dev dependency vulnerabilities (npm audit)

**Confidence:** 9/10  
**Phase:** Phase 3 — Dependency Supply Chain  
**Category:** OWASP A03  
**Location:** `package.json` / `package-lock.json`

**Description:**  
`npm audit` reports 5 vulnerabilities in dev/build toolchain (not shipped in `.app`):
- **vite** 8.0.3 — multiple dev-server path traversal / fs.deny bypass (HIGH in advisory, dev-only)
- **devalue** (via svelte) — DoS via sparse array (HIGH, build/runtime dep)
- **svelte** 5.55.1 — XSS/ReDoS advisories (mostly SSR/DOM clobbering; limited impact in CSR Tauri webview)
- **postcss**, **esbuild** — moderate/low, build/dev only

**Exploit scenario:** Developer runs `npm run tauri:dev` on untrusted network; attacker exploits Vite dev server. **Not end-user production risk.**

**Remediation:** `npm update vite svelte` (fixAvailable: true per audit); re-run `npm audit`.

**Priority:** P2 (fix this month)

### [INFO] FINDING-008: Document parser trust boundary

**Confidence:** 10/10  
**Phase:** Phase 1 / STRIDE  

Users intentionally parse untrusted documents (PDF, Office, images). LiteParse + ImageMagick/LibreOffice subprocesses are an expected **local code execution / DoS** surface. Mitigations: per-file 300s timeout in sidecar, worker semaphore, user-initiated only.
